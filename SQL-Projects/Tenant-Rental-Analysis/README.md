# Tenant Rental Analysis (SQL Project)

## Project Objective

Analyze tenant rental data to understand rental patterns, tenant segmentation, referral behavior, and property occupancy using SQL queries.

## Dataset Description

The dataset contains information related to tenants, houses, rent payments, and referrals.

Main attributes include:

* Tenant ID
* Tenant Name
* Contact Number
* City
* House ID
* Rent Amount
* Referral Details

This dataset helps analyze tenant behavior and property performance.

## Database Structure

Tables used in this project:

* tenants
* houses
* rent_payments
* referrals

An ER diagram is included in the repository to represent the relationships between these tables.

## Business Problems Solved

1. Classify tenants into customer segments based on rent paid.
2. Identify tenants who have never referred another tenant.
3. Find the house with the highest occupancy.
4. Calculate the average rent paid by tenants.
5. Identify top paying tenants.
6. Analyze referral patterns among tenants.
7. Calculate total rent revenue generated.
8. Rank houses based on occupancy.
9. Identify tenants paying above average rent.
10. Detect possible duplicate or missing data.

## SQL Concepts Used

* SELECT
* JOIN
* GROUP BY
* CASE Statements
* Subqueries
* CTE (Common Table Expressions)
* Window Functions
* Aggregate Functions

## Tools Used

* SQL Server Management Studio (SSMS)
* SQL

* GitHub

## Key Business Insights

* Certain houses generate higher rental revenue due to higher occupancy.
* A small group of tenants contributes significantly to total rent revenue.
* Referral activity among tenants is limited, suggesting potential for referral programs.
* Tenant segmentation helps identify high-value renters.

## Conclusion

This project demonstrates how SQL can be used to analyze rental data and generate business insights. The analysis highlights tenant segmentation, occupancy trends, and referral behavior that can help property managers make data-driven decisions.
